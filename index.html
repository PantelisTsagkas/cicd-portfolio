My CI/CD Pipeline Achievement

Automating Deployment from Code to Cloud

Overview

I'm excited to share a key achievement: successfully implementing a Continuous Integration and Continuous Deployment (CI/CD) pipeline. This project automates the process of deploying a simple Node.js web application from source control (GitHub) to an AWS EC2 instance, significantly streamlining the development and release cycle.

Key Components

GitHub (Source Control & Actions)

Node.js (Application Runtime)

AWS EC2 (Hosting Environment)

SSH (Secure Deployment)

Pipeline Flow

GitHub Push
Webhook Trigger
Test
Build
Deploy
Live

Process Details:

  1. Code changes pushed to GitHub.
  2. GitHub Actions workflow triggered automatically.
  3. Code is checked out, dependencies installed, and tests are run in a dedicated test job.
  4. Upon successful tests, the deployment job uses SSH to connect securely to the AWS EC2 instance.
  5. Latest code is pulled from the main branch onto the EC2 instance.
  6. Application dependencies are installed with npm ci for production reliability.
  7. Application is started or restarted using PM2 with health check verification.

Key Achievements & Benefits

  • Automated Deployment: Eliminates manual steps, reducing errors and increasing speed.
  • Faster Feedback Loop: Code changes are tested and deployed rapidly.
  • Improved Reliability: Consistent, repeatable deployment process with automated testing.
  • Secure Credential Management: Utilized GitHub Secrets for sensitive information.
  • Foundation for Scalability: Environment configuration ensures consistent Node.js runtime and process management.
